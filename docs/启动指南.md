# AI 报价 启动指南（新手向，macOS / Windows）

本指南假设你完全不会编程，只需照着一步步做即可。

## 方式一：直接安装（推荐）

不需要打开终端、不需要安装 Node.js，最简单，适合日常使用。

### macOS

1. 打开浏览器，访问：https://github.com/baotangyin/ai-quote-free/releases
2. 找到最新版本，下载 `ai-quote-*.dmg` 安装包：
   - 你的 Mac 是 **Apple 芯片**（M1/M2/M3/M4 等，2020 年之后的大多数机型），选文件名里带 `mac-arm64` 的那个。
   - 你的 Mac 是 **Intel 芯片**（较老的机型），选文件名里带 `mac-x64` 的那个。
   - 拿不准的话：点屏幕左上角苹果图标 →「关于本机」，看「芯片」或「处理器」一栏。
3. 下载完成后双击打开这个 `.dmg` 文件，会弹出一个窗口。
4. 把里面的「AI报价」图标拖到「应用程序（Applications）」文件夹图标上，等待复制完成。
5. 打开「访达」→「应用程序」，找到「AI报价」，双击打开。

**首次打开会被拦截，这是正常现象（因为这个安装包暂时没有经过苹果付费签名认证）：**

**情况一：提示「无法验证开发者」（标准流程）** — 这说明签名已生效，系统在进行安全检查，属于正常的未认证开发者应用：
1. 右键点击「应用程序」文件夹里的「AI报价」应用图标，选择「打开」。
2. 弹出确认框后再点一次「打开」即可启动。

这个步骤**只需要做一次**。以后直接双击「AI报价」图标打开即可，跟其他 Mac App 一样用。

**情况二：提示「已损坏，无法打开」（旧版本安装包）** — 这说明使用了之前未签名的旧版本安装包。请在终端执行以下命令解除限制：
```
sudo xattr -cr /Applications/AI报价.app
```
然后再双击「AI报价」应该能正常打开。如果还是不行，重新下载最新版本安装包重新安装。

> **说明**：这个提示是因为软件目前没有购买 Apple 开发者账号（每年 $99）做签名和公证，属于正常的自制小工具现象，不代表软件有问题。如果以后要正式对外销售或分发给不熟悉电脑操作的客户，建议购买 Apple Developer 账号做签名+公证，这样双击就能直接打开，不会再出现拦截提示。

### Windows

1. 打开浏览器，访问：https://github.com/baotangyin/ai-quote-free/releases
2. 找到最新版本，下载 `ai-quote-*-win-x64.exe` 安装包（文件名里带 `win` 和 `x64` 字样）。
3. 下载完成后双击这个 `.exe` 文件，会弹出安装向导，可以选择安装目录，一路点击「下一步」「安装」即可。

**双击安装时可能会被「Windows SmartScreen」拦截，这是正常现象（因为这个安装包暂时没有购买代码签名证书）：**

1. 会弹出一个蓝色窗口，写着「Windows 已保护你的电脑」，下面提示「未知发布者」。
2. 点窗口里的「更多信息」（More info）链接。
3. 窗口下方会出现「仍要运行」（Run anyway）按钮，点击它即可继续安装。

这个步骤**只需要在安装时做一次**，安装完成后桌面会出现「AI报价」图标，以后直接双击打开即可。

> **说明**：这个提示是因为软件目前没有购买代码签名证书，属于正常的自制小工具现象，不代表软件有病毒。如果以后要正式对外销售，建议购买代码签名证书对安装包签名，这样就不会再出现 SmartScreen 拦截提示。

如果你是开发者、需要修改代码或调试，请看下面的「方式二」。

## 方式二：开发者模式

以下方式需要用到「终端」，适合开发者或需要跑最新源码的场景。全程大约需要 10-15 分钟（首次安装可能更久，取决于网络）。

### 第一步：安装 Node.js

1. 打开浏览器，访问：https://nodejs.org/
2. 网站首页会有一个绿色按钮，写着「LTS」字样（LTS 代表长期支持版，是最稳定的版本），点击下载。下载下来的文件类似 `node-v20.x.x.pkg`。
3. 在「访达」（Finder）的「下载」文件夹里找到这个 `.pkg` 文件，双击打开。
4. 会弹出一个安装向导，一路点击「继续」「同意」「安装」，中途可能需要输入你的 Mac 开机密码（这是正常的系统权限确认，放心输入）。
5. 看到「安装成功」后，点击「关闭」，可以把安装包拖进废纸篓。

安装完成后，Node.js 就装好了，后面的步骤都会用到它。

### 第二步：打开「终端」

1. 按下键盘上的 `Command（⌘）` + `空格`，会弹出一个搜索框（Spotlight）。
2. 输入「终端」（或英文 `Terminal`），按回车打开。
3. 会看到一个黑色或白色背景的窗口，里面有一行文字加一个光标在闪烁，这就是终端，可以在这里输入命令。

**小提示**：终端里输入命令后，按一次 `回车（Return）` 键，命令才会真正执行。

### 第三步：安装项目依赖

在终端里，逐字输入下面这一整行命令（可以直接复制粘贴），然后按回车：

```
cd ~/AI\ pricing/ai-quote && npm install
```

说明：
- `cd ~/AI\ pricing/ai-quote` 的意思是「进入 AI pricing/ai-quote 这个文件夹」。如果你的项目文件夹路径不是这个，请把这部分换成你实际的路径（可以把文件夹从「访达」直接拖进终端窗口，终端会自动帮你填好路径）。
- `npm install` 会自动下载这个项目需要用到的所有工具包，**其中包含 Electron（用来打包成桌面应用的工具），体积较大，第一次安装可能需要几分钟**，请耐心等待，不要中途关闭终端窗口。
- 安装过程中如果弹出对话框要求输入 Mac 密码，或者终端里提示需要密码，**不用害怕**，这是系统在确认安装权限，正常输入你的开机密码即可（输入时屏幕上不会显示字符，属于正常现象）。
- 如果看到一堆黄色的 `npm warn` 警告文字，一般可以忽略，只要最后没有出现红色的 `error` 就说明成功了。

### 第四步：启动开发版程序

依赖装好之后，在终端里继续输入：

```
npm run dev
```

按回车。稍等几秒到十几秒，会自动弹出一个桌面应用窗口，这就是「AI 报价」软件本身。

以后每次想打开软件，都重复「第二步」打开终端 + 「第四步」这一条命令即可（第三步的 `npm install` 只需要做一次，除非后面重新拿到了新版代码）。

**如何关闭软件**：直接关闭弹出的应用窗口即可；终端里可以按 `Control（⌃）+ C` 停止后台运行，或者直接关闭终端窗口。

## 数据保存在哪里？如何备份？

软件里录入的所有数据（产品库、供应商、报价、项目）都保存在一个数据库文件里，位置固定在：

```
~/Library/Application Support/ai-quote/ai-quote.db
```

也就是「用户主目录 → Library → Application Support → ai-quote → ai-quote.db」。

**如何打开这个位置查看**：
1. 打开「访达」（Finder）。
2. 按 `Command（⌘）` + `Shift（⇧）` + `G`，会弹出一个「前往文件夹」输入框。
3. 输入 `~/Library/Application Support/ai-quote` 并按回车，就能看到 `ai-quote.db` 文件。

**如何备份**：定期把 `ai-quote.db` 这个文件复制一份到别的地方（比如移动硬盘、云盘、U盘）即可，文件名可以改成带日期的，例如 `ai-quote-2026-07-08备份.db`。如果以后软件出问题需要恢复，把备份文件改回原名 `ai-quote.db` 放回原位置覆盖即可。

## 常见问题

**1. 启动时提示 `better-sqlite3` 相关报错（例如提到 `NODE_MODULE_VERSION` 或 `bindings`）**

这是数据库底层组件版本不匹配导致的（正常情况下 `npm install` 时会自动重建好，一般不需要手动处理；如果还是遇到了），在终端里进入项目目录后执行：

```
cd ~/AI\ pricing/ai-quote && npm run rebuild:electron
```

这一步是「为应用运行环境重建数据库组件」，即针对 Electron（软件运行时使用的环境）而不是普通 Node.js 重新编译 `better-sqlite3`。等待执行完成后，再重新运行 `npm run dev`。

**1.1 首次安装时弹出「需要安装 Xcode Command Line Tools」提示**

这是 macOS 系统在提示需要一个编译工具，属于正常现象。点击弹窗里的「安装」，耐心等待安装完成（可能需要几分钟到十几分钟，取决于网络），完成后回到终端重新执行一次「第三步」的 `npm install` 命令即可。

**1.2 国内网络较慢，`npm install` 一直卡住或很慢**

可以先切换到国内镜像源，再重新安装：

```
npm config set registry https://registry.npmmirror.com
```

执行完这一行后，再重新执行「第三步」的 `npm install`。

**2. 提示端口被占用（例如 `EADDRINUSE` 或「端口已被使用」）**

说明上一次的程序可能还在后台运行。可以先关掉所有终端窗口，重新打开一个新的终端窗口，再执行「第三步」和「第四步」。如果还是不行，重启一下电脑，再重新启动软件。

**3. 想要彻底重来（怀疑装错了、乱七八糟报错很多）**

在终端里执行：

```
cd ~/AI\ pricing/ai-quote && rm -rf node_modules && npm install
```

这会删除已经下载的所有依赖包，重新完整下载安装一遍，相当于「重装」，同样需要等待几分钟。**注意：这个操作不会影响到你已经录入的数据（数据库文件是单独存放的，见上文「数据保存在哪里」），可以放心执行。**

**4. 命令输错了怎么办**

终端里的命令必须完全按照文档中的写法输入（包括空格、斜杠、`&&` 符号），建议直接复制粘贴而不是手动敲。如果报错，把报错的完整文字截图，找技术支持的人帮忙看一下即可，不用自己瞎改。

## 面向开发者的补充说明（普通用户可跳过本节）

`better-sqlite3` 是一个需要针对具体运行环境编译的原生模块，Electron 运行时和系统 Node.js 运行时用的是不同的编译版本（ABI），两者不能通用：

- `npm install` 执行后会自动尝试把 `better-sqlite3` 重建为 **Electron 版本**（通过 `postinstall` 钩子调用 `npm run rebuild:electron`），这样 `npm run dev` 才能正常启动应用。
- 如果你是开发者，装完依赖后想跑 `npm test`（使用系统 Node.js 执行），需要先执行 `npm rebuild better-sqlite3` 把它重建回 **Node.js 版本**，测试完如果要继续跑应用，再执行 `npm run rebuild:electron` 切回 Electron 版本。
- 普通用户走「第三步 → 第四步」（`npm install` → `npm run dev`）这条主流程即可，全程不需要关心这一细节，`postinstall` 会自动处理好。

## 附录：报价单导入功能验收步骤

用于验收「报价单导入」这个功能是否正常好用，照着下面的顺序走一遍即可，全程不需要懂代码。

### 第一步：配置 AI 接口

1. 打开软件，点左侧菜单「设置」。
2. 找到「AI 识别配置」这一块，依次填好：
   - **协议**：选 `openai` 或 `anthropic`，取决于你用的是哪家的接口（大多数中转/自建接口选 `openai` 协议即可）。
   - **Base URL**：AI 服务商提供的接口地址。
   - **API Key**：AI 服务商提供的密钥（只会保存在本机数据库里，不会上传到别处）。
   - **模型名**：要使用的模型名称，例如 `gpt-4o-mini`、`claude-3-5-haiku-20241022` 等（具体填什么以你的服务商文档为准）。
3. 点「保存」，再点旁边的「测试连接」，看到提示连接成功即可继续；如果失败，检查 Base URL / API Key / 模型名是否填错。

### 第二步：导入 samples 里的供应商报价单

1. 点左侧菜单「报价单导入」，进入导入向导。
2. 第一步「选文件」：点「选择报价单文件」，选中项目文件夹里 `samples/` 目录下的任意一份真实供应商报价单（例如 `2026年最新报价.xls`），确认下方列出了解析出的表格分块，勾选要导入的分块，点「下一步」。
3. 第二步「识别」：点「开始识别」，等待 AI 依次识别每个分块，识别完成后会提示「识别完成：共识别 N 行；X 行无法识别已跳过；Y 块解析失败」。

### 第三步：核对

1. 进入第三步「核对」，逐行检查表格识别出的产品名称、品牌、型号、单价、选配项是否与原始报价单一致。
2. 单价明显异常或置信度较低的行会有底色提示，可手动编辑更正，也可以删除识别错误的整行。
3. 确认无误后点「下一步」。

### 第四步：入库

1. 第四步「关联供应商」：选择（或新建）本次报价单对应的供应商，点「下一步」。
2. 第五步「入库」：点「开始入库」，等待提示「入库完成」，并会显示本次新建了多少个产品、更新了多少条价格记录。

### 第五步：在产品库里核对入库结果

1. 点左侧菜单「产品库」。
2. 用搜索框按名称/品牌/型号找到刚才导入的产品，检查分类、名称、品牌、型号、参数、尺寸、选配加价项是否正确。
3. 点进产品详情查看价格记录面板，确认价格来源标注为对应的供应商，金额与原始报价单一致（注意：报价单里如果写的是「元」，软件内部会自动换算成「分」入库，界面上看到的金额仍然是「元」，属于正常显示）。
4. 如果报价单里同一产品用「万元」计价（如「1.5万」），核对入库价格是否等于「万元数 × 10000」换算后的元数（例如 1.5万 = 15000 元）。
