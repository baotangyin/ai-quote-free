# AI 报价单（免费版）

面向展厅 / 智能化集成行业的桌面报价软件。用 AI 把报价单识别入库、把平面图纸变成项目清单，围绕"产品库 → 项目 → 导出"打通从询价到交付文档的完整链路。

基于 Electron + React + TypeScript + better-sqlite3 构建，数据全部存储在本地，支持 macOS 与 Windows。

## 下载安装

在 [Releases](../../releases) 页面下载最新版本：

- **Windows**：`*.exe`（NSIS 安装包）
- **macOS**：`*.dmg`。安装包为 ad-hoc 签名，首次打开如提示"无法验证开发者"，请在包上右键 → 打开，或执行 `xattr -cr /Applications/AI报价单.app`

**自动更新**：「设置 → 软件更新」可查看当前版本、手动检查更新，并选择更新模式——「自动下载并提示重启安装」或「仅提示新版本，手动打开下载页」。**mac 因安装包未签名不支持自动安装**，只能手动下载安装包重新安装；Windows 默认走自动模式。

## 核心功能

### 产品库与供应商
- 产品多分类标签、选配项（选配自动联动参数与加价）、核心/招标/投标三套参数、推荐品牌、技术指标（功耗/机柜 U 数/网口等）
- **类别参数模板**：按分类预置一套默认技术参数（单位/用电量/机柜 U 数/网口/com口 + 核心/招标/投标三套参数文本），手工新建产品与 AI 报价单导入落库时按所属分类自动补齐空值字段，不覆盖已填内容
- 价格记录按供应商沉淀，成本取值规则可配（最低价 / 最新价 / 指定供应商），支持产品级覆盖
- 供应商管理（名称/联系人/电话/地址/付款方式/开户信息/备注）
- **供应商询价单**：项目清单行可按选定供应商各生成一张不含我方价格的询价单，供应商页填写回价后一键「写入价格记录」，形成询价 → 回价 → 入库闭环；支持导出询价单为 Excel
- 产品/项目等五大列表统一支持多选、深度筛选与批量操作

### AI 报价单导入
- 直接导入供应商 Excel 报价单（xls/xlsx，兼容 WPS、并排表格、备注加价、矩阵定价等真实格式），AI 识别为结构化产品行
- 自动匹配产品库：已有产品更新价格，新品一键入库；识别结果逐行人工核对可改
- 大文件自动分块、截断抢救、失败块跳过，识别过程可离开页面不丢进度

### AI 图纸识别
- 上传平面布置图（png/jpg/webp/PDF，PDF 自动逐页转图），AI 识别图上的空间名称与设备标注（含类别、尺寸、数量，如"65寸电容触控一体机*2"）
- 识别结果人工核对：改名、调数量、删行、换绑产品库产品或设为手工行
- 一键生成到已有项目的指定板块，或直接从图纸新建项目（自动套用项目类型模板）
- dwg 请先从 CAD 导出 PDF/图片

### 项目报价（四层结构：项目 → 板块 → 空间 → 清单行）
- 四种模式：**概算**（项目总投资估算表，手填/系数/引用板块三种取值）、**方案预算**、**造价清单**、**投标**
- 清单行快照机制：建行时锁定产品参数与成本，产品库变动不影响已报项目，支持一键刷新
- 行级倍率、手动定价、空间小计、系统集成费、置底空间（安防监控/中控网络等恒排板块末尾）
- **多供应商比价**：每行并联多个候选成本，选定生效并导出成本对比版
- **联动规则引擎**：按产品/分类/项目类型触发配套推荐（如 LED 屏自动算接收卡/电源数量），数量公式安全求值，弹面板勾选加入不静默改单
- **项目类型模板**：按类型（如"展厅"）预置板块+空间骨架，新建项目自动生成
- 项目深复制、批量导出

### 导出
- 一次导出多版本 xlsx：含成本完整版 / 对外报价版 / 实施清单（含汇总表、跨 sheet 公式、技术指标合计）
- **导出模板**：列结构、表头样式、公司抬头/落款、汇总行与税金行均可配置（多套「版本集」对应不同导出版本）；支持导入客户的 xlsx 由 AI 识别列结构自动转为模板
- **自定义列**：每个版本除系统列外可新增任意数量的自定义列（列名/宽度/固定内容，同一版本下每行填充相同固定文本），不参与系统列的模式列集判断，适合「供货范围」「质保期」等与产品无关的固定说明列
- 概算模式导出项目总投资估算表（万元、活公式）

### 统计分析
- 总览：项目数/清单行数/成本/报价/利润/利润率统计与按月趋势图
- 产品利润排行、单产品价格趋势图（供应商报价/AI查价/手动三类来源）
- 价格异动：按查价结果展示涨幅榜/跌幅榜，异动幅度可配置阈值提醒
- 利润口径基于清单行的固化快照成本计算，不随产品库价格实时变动

### 价格监控与定时查价
- 产品可标记「监控」，按设定周期（每日/每周/每月）由 AI 自动查价并写入价格记录，超阈值涨跌记为异动
- 查价模型需支持联网搜索，可与识别用模型分开配置；支持随时手动「立即查价」
- 关闭主窗口可最小化到系统托盘后台运行，托盘菜单支持「打开主界面」「立即查价」「退出」
- 开机自动启动（mac / Windows，默认关闭，Linux 暂不支持）

## AI 配置

首次使用请在 **设置** 页新增至少一个 **AI 配置档案**（识别功能需要），采用档案制：可按需新增多个不同厂家/账号的接口档案，各自独立配置与测试：

| 配置项 | 说明 |
|---|---|
| 名称 | 档案备注名，便于在多个厂家/账号间区分 |
| 协议 | `openai`（兼容 OpenAI Chat Completions 的任意服务）或 `anthropic` |
| Base URL | 服务地址 |
| API Key | 密钥（仅存本地 SQLite） |
| 模型 | 该档案使用的模型名 |

每个档案可单独「测试连接」验证。新建/维护好档案后，再分别为三种用途绑定档案：

| 用途 | 说明 |
|---|---|
| 文本识别 | 报价单导入识别、导出模板 AI 转换等文本类任务 |
| 图片处理 | 图纸识别等图片输入任务，需绑定支持视觉（图片输入）的模型档案 |
| 定时查价 | 需绑定支持联网搜索获取实时价格的模型档案（设置 → 查价监控配置） |

三种用途可共用同一档案，也可分别指定不同厂家的档案；未单独绑定图片处理 / 定时查价用途时默认与文本识别共用同一档案。删除已被绑定的档案时，对应用途会自动回退绑定到剩余档案中的第一个并提示。

除 AI 识别 / 查价外，其余功能均可离线使用。

## 开发

```bash
npm install
npm run dev        # 开发模式（electron-vite）
npm test           # vitest 全量测试
npm run typecheck  # tsc --noEmit
npm run build      # 构建
```

### 代码结构

```
src/
  core/      纯 TS 业务核心（db/repo/domain/export/import/ai），不依赖 Electron
  main/      Electron 主进程 + IPC
  preload/   预加载脚本（invoke 映射）
  renderer/  React UI（不直接引用 core，类型经 src/shared/api-types）
  shared/    进程间共享的 API 契约类型
tests/       vitest（core/main/renderer 分层）
```

- 金额一律以"分"存整数，仅展示层转元/万元
- 数据库 schema 变更走 `PRAGMA user_version` 增量迁移，升级不丢存量数据
- 推送 tag 后 GitHub Actions 自动构建 Mac/Windows 安装包并发布 Release
